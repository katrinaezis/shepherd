If you want to add another step..


tour.addstep('start', {
	title: 'Adding another step with classes',
	text: 'you can add classes to change theme',
	attachTo: 'element right',
	classes: 'shepherd-theme-arrows',
	buttons: [
		{
			text: 'Next',
			action: tour.next
		}
	]
});
  • attachTo: What element the step should be attached to on the page. For example: ".some #element left". If you don't specify an attachTo the element will appear in the middle of the screen.
  • classes: Extra classes to add to the step. shepherd-theme-arrows will give you our theme.

    • If you want to add another button..

    
tour.addstep('second', {
	title: 'Lets try adding a step with 2 buttons',
	text: 'you can simply add buttons by inserting more inside the step button method',
	attachTo: 'element bottom',
	classes: 'shepherd-theme-arrows',
	scrollTo: true
	buttons: [
		{
			text: 'Back',
			classes: 'shepherd-button-secondary',
			action: tour.back
		},
		{
			text: 'Next',
			action: tour.next
		}
	]
});
  • buttons: An array of buttons to add to the step. By default we add a Next button which triggers next(), set this to false to disable. Each button in the array is an object of the format:
    1. text: The HTML text of the button
    2. classes: Extra classes to apply to the <a>
    3. action: A function executed when the button is clicked on
    4. events: A hash of events to bind onto the button, for example {'mouseover': function(){}}. Adding a click event to events when you already have an action specified is not supported.
  • scrollTo: Should the element be scrolled to when this step is shown?
