The API

Shepherd provides an extensive API and documentation that explains it's behavior. Let's go through some useful calls.

Tour Instances

We can add options to our tour, but first we have to create the tour.

myTour = new Shepherd.Tour({ options });
Now, we are going to look at how we can work with this instance. steps and defaults are the options of the tour instance. Its methods are as follows.
  • addStep(id, options) - As we saw in our previous lessons, a step is created by assigning an ID to it, then adding options such as text or buttons, which will be described later in this lesson.
  • getById(id) - This method is used to select any particular step by its ID.
  • show(id) - Show a particular step by ID.
  • on(event, handler) - Binds an event to your tour. This is similar to jQuery's bind() method.
  • off(event, handler) - Unbinds an event
A tour instance also has events like start, complete, show, and hide.

Steps

Let's look at the options you can add to steps. All of these have been reviewed in your lessons
  • title - Add a title
  • text - The text to go along with a step
  • attachTo - Two parts: the selector of the element where the step is to be attached, and the position to attach the step to. We went over this in lesson two and three.
  • classes - Extra classes to add to your dialog.
  • buttons - The list of buttons to be shown
Methods to make your life easier
  • show() - Show a step
  • hide() - Hide a step
  • cancel() - Hide step and cancel the tour
  • complete() - Hide step and complete tour
  • destroy() - Destroys a step
  • on(event, handler) - Binds an event
  • off(event, handler) - Unbinds an event

That's all folks!
